# Reglamento Oficial Programming Battle 2026

---

## Reglas generales
1. Todos los participantes deben registrarse previamente.
2. Los equipos deberán respetar horarios.
3. No se permite compartir código entre equipos.
4. No usar internet.
5. No IA generativa.
6. No compartir código.
7. Entregas fuera de tiempo no cuentan.
8. Solo software autorizado.
9. Se permite usar apuntes.

---

## Penalizaciones
- Copia: descalificación
- Retraso en entrega: penalización
- Conducta inapropiada: expulsión

---

# Evaluación

## Criterios
| Criterio | Porcentaje |
|---|---|
| Funcionamiento | 40% |
| Lógica | 20% |
| Eficiencia | 15% |
| Limpieza de código | 15% |
| Presentación | 10% |

---

# Entregas
- Cada reto deberá entregarse antes del tiempo límite.
- Se recomienda comprimido (zip, rar).

---

# Desempates
1. Mayor número de ejercicios resueltos.
2. Menor tiempo total.
3. Mejor eficiencia.
